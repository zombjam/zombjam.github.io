[筆記] JavaScript的Hoisting

2019-08-26 | JavaScript

紀錄一下 Hoisting 的觀念與使用。
一般在寫程式的時候，我們都會定義好變數，才會去使用它。
然而在尚未定義的情況下，直接去使用這個變數，通常都會出現錯誤訊息！

首先，先來看 w3schools 跟 MDN 對於 Hoisting 的說明

w3school 的說明

Hoisting is JavaScript’s default behavior of moving declarations to the top.

MDN 的說明

提升(Hoisting)是一種釐清 JavaScript 在執行階段內文如何運行的思路(尤其在創建和執行階段)。
然而，提升一詞可能會引起誤解：例如，提升看起來是單純地將變數和函式宣告，移動到程式的區塊頂端，然而必非如此。變數和函式的宣告會在編譯階段就被放入記憶體，但實際位置和程式碼中完全一樣。

意思是在 JavaScript 中，它會把定義的變項一到最前面先執行。
