npm 套件管理工具

2019-06-02 | 工欲善其事

npm 的全名為 Node Package Manager，是附屬在 Node.js 中的套件管理工具。
在安裝 Node.js 時，就可以順便將 npm 安裝到電腦中。

常用語法

  • npm doc

dependencies與devDependencies的差別

npm install 的時候，有兩種指令把他們寫入 package.json 文件裡面
–save：寫入 dependencies
–save-dev：寫入 devDependencies

兩者差異：

dependencies 是需要發佈到生產環境的套件。
devDependencies 只用於開發環境。

※ dev 是 develop 的簡寫。

package-lock.json 的作用

package-lock.json 為 npm 5 新增的檔案
每次 npm install/uninstall/undate 時，都會去更新

用以紀錄當前狀態下實際安裝的各個 npm package 的具體來源與版本號。

版本號

npm 遵守semver語意化版本號的規則，使用 node-semver 來解析版本號。

版本號的格式

一般來說，一個版本號包含三個部分：

MAJOR 表示主版本號，MINOR 表示次版本號，PATCH 表示修訂號。
它們的取值為非負整數，若大於零，不能在前方補零。
另外，版本號中任何位置都不能有空格。

如何升級版本號

升級版本號的規則主要有三個：

  1. 主版本號：做了不兼容的 API 修改
  2. 次版本號：做了向下兼容的功能性新增
  3. 修訂號：向下兼容的問題修正

版本號符號意思

^

表示同一主版本號中，不小於指定版本號的版本。
例如：^2.2.1 對應主版本號為 2，不小於 2.2.1 的版本號，比如：2.2.1、2.2.2、2.3.0。

~

表示同一主版本和次版本號中，不小於指定版本號的版本
例如：~2.2.1 對應主版本號為 2，次版本號為 2，不小於 2.2.1 的版本號，比如：2.2.1、2.2.2。

>、<、=、>=、<=、-

用來指定一個版本號的範圍。
※ 注意使用 - 的時候，前後必須有空格。
